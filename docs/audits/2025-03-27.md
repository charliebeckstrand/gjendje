# Audit: 2025-03-27

Full audit of the gjendje state management library (v1.3.0). Covered all source files in `src/`, including core state classes, adapters, primitives, framework bindings, and supporting utilities.

---

## Critical

### 1. React hook causes unnecessary resubscriptions on every render

- **File:** `src/react/index.ts`
- **Problem:** Both `getSnapshot` and the `subscribe` callback passed to `useSyncExternalStore` are recreated on every render. React compares `subscribe` by reference â€” a new closure each render triggers teardown and resubscription every cycle, causing O(renders) subscription churn.
- **Resolution:** Memoize `subscribe` with `useCallback` (dep: `[instance]`) and `getSnapshot` with `useCallback` (dep: `[instance, selector]`).
- [x] Patched

### 2. Storage adapter bypasses validation when custom serializer is used

- **File:** `src/adapters/storage.ts`
- **Problem:** When `options.serialize` is provided, `parse()` calls `serialize.parse(raw)` directly, bypassing `readAndMigrate()` entirely. This means `validate`, `migrate`, and `version` options are silently ignored. A user providing both `serialize` and `validate` gets no validation on reads from storage.
- **Resolution:** After custom serializer parse, still run the validation step. Report `ValidationError` and fall back to default on failure.
- [x] Patched

### 3. Batch flush loop has no infinite loop protection

- **File:** `src/batch.ts`
- **Problem:** The `flush()` function uses `while (queue.length > 0)` with no iteration cap. Circular reactive dependencies can freeze the browser/app since each flush iteration increments the generation counter, allowing the same notification to be re-queued indefinitely.
- **Resolution:** Add a `MAX_FLUSH_ITERATIONS = 100` guard. Log error and clear queue if exceeded.
- [x] Patched

## High

### 4. `effect()` errors bypass the global error pipeline

- **File:** `src/effect.ts`
- **Problem:** Effect cleanup errors and callback errors use `console.error` directly instead of `reportError()` from the config system. Users who set up `configure({ onError })` never see effect errors.
- **Resolution:** Import `reportError` and route errors through it alongside the existing `console.error`. Accept optional `key` in effect options for error attribution.
- [x] Patched

### 5. `configure()` cannot clear callbacks or reset to defaults

- **File:** `src/config.ts`
- **Problem:** `globalConfig = { ...globalConfig, ...config }` â€” spreading `undefined` values does NOT delete existing keys. Once a callback like `onError` is set, there's no way to clear it. Also no `resetConfig()` for testing or HMR.
- **Resolution:** Iterate entries and explicitly delete keys set to `undefined`. Add and export `resetConfig()`.
- [x] Patched

### 6. URL adapter always uses `pushState`, creating excessive history entries

- **Files:** `src/types.ts`, `src/adapters/url.ts`
- **Problem:** Every `set()` call pushes a new browser history entry. For rapid state changes (e.g., search-as-you-type, sliders), this creates dozens/hundreds of entries, making back-button navigation unusable.
- **Resolution:** Add `urlReplace?: boolean` option to `StateOptions`. When true, use `replaceState` instead of `pushState`.
- [x] Patched

### 7. No global cleanup / `destroyAll()` utility

- **Files:** `src/registry.ts`, `src/index.ts`
- **Problem:** The registry tracks all instances but provides no bulk teardown. In testing, HMR, or SPA route transitions, stale instances accumulate with active subscriptions and storage event listeners.
- **Resolution:** Add `destroyAll()` that iterates the registry, destroys each instance, and clears the map. Export from index.
- [x] Patched

## Medium (not patched this session)

### 8. `select()` pays full `computed()` overhead for single dependency

- **File:** `src/select.ts`
- **Problem:** `select()` wraps a single source in an array and delegates to `computed()`, paying for array allocation, dependency loop, and Promise.all checks â€” all unnecessary for a single dependency.
- **Resolution:** Implement `select()` as a lightweight standalone function without multi-dep machinery.
- [x] Patched â€” standalone implementation, ~13% faster ops, no array allocation

### 9. `shallowEqual` doesn't handle `Date`, `RegExp`, `Set`, `Map`

- **File:** `src/utils.ts`
- **Problem:** `shallowEqual(new Date(0), new Date(0))` returns `false` (falls through to reference equality). Same for RegExp. Users relying on `shallowEqual` for hydration comparisons or `isEqual` may hit false negatives.
- **Resolution:** Add special cases for `Date` (compare via `getTime()`), `RegExp` (compare `toString()`), `Set` (compare size + elements), and `Map` (compare size + entries).
- [x] Patched in 2026-03-27 data integrity audit

### 10. Code duplication between `StateImpl` and `MemoryStateImpl`

- **File:** `src/core.ts`
- **Problem:** ~300 lines of duplicated logic across `set()`, `reset()`, `patch()`, `intercept()`, `onChange()`, `watch()`, and `destroy()`. Per project constraints, merging the classes is prohibited for performance reasons. However, bugs fixed in one class can be missed in the other.
- **Resolution:** Add a behavioral parity test suite that runs identical scenarios against both memory and non-memory scopes and asserts identical outcomes.
- [x] Addressed â€” 83 behavioral parity tests across both scopes guard against drift
